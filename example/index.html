Micro framework following atomic design.

It's build from atoms

As Brad Frost said "Atomic design is methodology for creating design systems". By definition the atom is a basic unit of matter. In the context of OrganicCSS the atom is simply a property and its value. For example:

font-size: 20px;
That's the smallest piece of your CSS code. Isn't it?

There are some predefined atoms

The atoms are the thing which will be transfered from project to project. This means that you really don't need to define them again and again. OrganicCSS will take care about this. The following snippet shows how to include atoms into your selector.

.login-box {
@include atoms((
block,
margin-top-20,
font-size-14,
bold
));
}
